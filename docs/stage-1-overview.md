# Stage 1 - LLM 应用基础

## 阶段目标

把 LLM 调用从“能跑 demo”升级成“后端服务里可维护的能力”。

完成后应该能做到：

- 统一封装模型调用
- 处理 timeout、retry、rate limit
- 使用结构化输出
- 校验模型返回结果
- 定义和执行简单 tool calling
- 记录 token、latency、error、cost

## 核心概念

### LLM API 工程化

把模型当作一个外部服务，而不是魔法函数。

需要关注：

- 请求超时
- 错误重试
- 限流
- token 统计
- 成本
- 日志脱敏

### 结构化输出

让模型返回程序能消费的 JSON，而不是自然语言段落。

常用工具：

- JSON Schema
- Pydantic
- Zod

### Tool Calling

让模型选择工具和生成参数，程序负责校验、执行、审计。

第一阶段先做只读或低风险工具：

- 搜索文件
- 读取文件
- 生成摘要
- 保存本地结果

## 阶段产出

- 一个最小 LLM 网关服务
- 一个 Markdown 笔记结构化摘要脚本
- 一份日志字段设计
- 一份阶段复盘
